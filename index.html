Introducción al Desarrollo Web

El desarrollo web es el proceso de diseñar, construir y mantener sitios o aplicaciones que funcionan a través de Internet o de una intranet. Implica tanto la parte visual (lo que ve el usuario) como la lógica interna (procesos y almacenamiento de datos).

En el mundo del desarrollo web, comprender cómo funciona la comunicación entre cliente y servidor es esencial. Desde este modelo se desprenden conceptos clave como los puertos de comunicación, el servidor web, el funcionamiento de una aplicación web y el despliegue de contenido en internet.

  • Modelo cliente-servidor

    El modelo cliente-servidor es la base de las aplicaciones web modernas.

    • Cliente: Normalmente es el navegador (Chrome, Firefox, Edge, Safari), que envía solicitudes al servidor.

    • Servidor: Es el equipo que recibe esas solicitudes, las procesa y devuelve una respuesta al cliente.

    Ejemplos:

    • Correo electrónico: el cliente (como Gmail u Outlook) se conecta al servidor de correo, el cual guarda y distribuye los mensajes. Cuando envías un correo, el cliente lo entrega al servidor, y cuando recibes, el cliente descarga el mensaje desde ese servidor.

    • Cuando escribes www.tienda.com en tu navegador, el cliente envía la petición al servidor, este busca los datos y responde con la página web que ves en pantalla.

  • Puertos de comunicación

    En esta comunicación entran en juego los puertos, que son canales virtuales para identificar diferentes servicios de red, gracias a estos puertos, el sistema sabe qué tipo de comunicación está manejando.

    • Puerto 80 (HTTP): tráfico web sin cifrado.
    • Puerto 443 (HTTPS): tráfico web seguro con certificado SSL/TLS.
    • Otros puertos comunes: 21 (FTP), 25 (SMTP), 3306 (MySQL).

  • Servidor Web

    • Un servidor web es el software (y hardware) encargado de recibir solicitudes del cliente y devolver los recursos necesarios: páginas HTML, imágenes, archivos CSS o respuestas JSON.

    Algunos de los más utilizados son:

    • Apache: Muy popular y flexible, soporta módulos para ampliar sus funcionalidades.
    • Nginx: Conocido por su alto rendimiento y capacidad para manejar muchas conexiones simultáneas.
    • Microsoft IIS: Integrado en sistemas Windows, ideal para entornos .NET.

    Sin un servidor web, los navegadores no podrían recibir la información que despliegan a los usuarios.

  • Funcionamiento de una Aplicación Web

    • El funcionamiento de una aplicación web se basa en la interacción constante entre el cliente (normalmente el navegador del usuario) y el servidor, a través de solicitudes y respuestas. El proceso puede describirse en pasos:

    1. El usuario accede a la aplicación desde su navegador o dispositivo móvil, ingresando una URL o haciendo clic en un enlace.
    2. El navegador envía una solicitud (request) al servidor web utilizando el protocolo HTTP o HTTPS.
    3. El servidor recibe la solicitud y la procesa. Si necesita información adicional, consulta a una base de datos (por ejemplo, la lista de productos, datos de usuarios o historial de compras).
    4. El servidor genera una respuesta (response), que puede ser una página HTML, un archivo JSON o cualquier recurso solicitado (imágenes, estilos, scripts).
    5. El navegador interpreta la respuesta y la presenta de manera visual al usuario.

    Ejemplo: Cuando entras a una tienda en línea y buscas un producto, el navegador envía tu búsqueda al servidor, este consulta la base de datos, obtiene los resultados y devuelve una página con los productos que coinciden con tu búsqueda.

  • Despliegue de Contenido Web
      • Desarrollo local: Creación y prueba en una computadora personal.
      Servidor de pruebas (staging): Un entorno para verificar que todo funcione bien.
      Producción: Publicación en un servidor web accesible al público.
      Dominio y DNS: Configuración de un nombre de dominio (ejemplo: www.miweb.com) que apunte al servidor.
      Mantenimiento: actualizaciones, correcciones y mejoras periódica.

Fundamentos de HTML

El Lenguaje de Marcado de Hipertexto (HTML) es el código que se utiliza para estructurar y desplegar una página web y sus contenidos. Por ejemplo, sus contenidos podrían ser párrafos, una lista con viñetas, o imágenes y tablas de datos. Como lo sugiere el título, este artículo te dará una comprensión básica de HTML y cuál es su función.

El propósito principal de HTML es:

  • Dar estructura y semántica al contenido (títulos, párrafos, listas, tablas, imágenes, enlaces).
  • Permitir que los navegadores interpreten y muestren la información de manera correcta.
  • Servir como base para integrar otros lenguajes como CSS (diseño) y JavaScript (interactividad).

Doctype: En HTML, el doctype es el preámbulo obligatorio <!DOCTYPE html> que se encuentra al principio de todos los documentos. Su único propósito es evitar que el navegador entre en el llamado "modo peculiar" al renderizar un documento; es decir, el doctype garantiza que el navegador haga todo lo posible por seguir las especificaciones pertinentes, en lugar de usar un modo de renderizado diferente que sea incompatible con algunas especificaciones.

Document Tree (DOM): Un "documento tree HTML" es una representación jerárquica de un documento HTML como un árbol de nodos, conocido como Modelo de Objetos de Documento (DOM). Este árbol, con un nodo raíz (el documento), nodos hijos (elementos, atributos, texto) y relaciones padre-hijo, permite a JavaScript acceder y modificar el contenido, la estructura y el estilo de una página web. Los navegadores utilizan el DOM para interpretar y mostrar el contenido de un sitio web.

Etiquetas comunes de HTML:

<p>: Define un párrafo.

<br>: Inserta un salto de línea.

<hr>: Inserta una línea horizontal para separar secciones.

<blockquote>: Define una cita textual larga.

<q>: Define una cita corta dentro de una oración.

<abbr>: Abreviatura con texto explicativo al pasar el mouse.

<cite>: Título de una obra (libro, película, etc.).

<mark>: Texto resaltado o destacado.

<small>: Texto con menor importancia.

<strong>: Texto con énfasis fuerte (negrita).

<em>: Texto enfatizado (cursiva).

<b>: Texto en negrita sin valor semántico.

<i>: Texto en cursiva sin valor semántico.

<u>: Texto subrayado.

<del>:Esta etiqueta se utiliza para texto tachado.

<sub>:esta etiqueta se utiliza para texto en subíndice.

<sup>:esta etiqueta se utiliza para texto en superíndice.

<a>: esta etiqueta se utiliza para crear un enlace a otra página o recurso. Usa el atributo href.

<img>: Inserta una imagen. Necesita atributos src y alt.

<iframe>: Inserta otra página web dentro del documento actual.

<ul>: Crea una Lista desordenada con viñetas.

<ol>: Crea una Lista ordenada numerada.

<dl>: Crea una Lista de definición.

<div>: Crea un Contenedor en bloque para agrupar contenido.

Fundamentos de CSS

CSS (Cascading Style Sheets) se usa para embellecer el HTML. Permite aplicar colores, fuentes, márgenes, animaciones y estilos responsivos.

h1 { color: #ff69b4; font-family: 'Comic Sans MS'; }
p { background-color: #f0f8ff; padding: 10px; }
