Introducción al Desarrollo Web

El desarrollo web es el proceso de diseñar, construir y mantener sitios o aplicaciones que funcionan a través de Internet o de una intranet. Implica tanto la parte visual (lo que ve el usuario) como la lógica interna (procesos y almacenamiento de datos).

En el mundo del desarrollo web, comprender cómo funciona la comunicación entre cliente y servidor es esencial. Desde este modelo se desprenden conceptos clave como los puertos de comunicación, el servidor web, el funcionamiento de una aplicación web y el despliegue de contenido en internet.

  • Modelo cliente-servidor

    El modelo cliente-servidor es la base de las aplicaciones web modernas.

    • Cliente: Normalmente es el navegador (Chrome, Firefox, Edge, Safari), que envía solicitudes al servidor.

    • Servidor: Es el equipo que recibe esas solicitudes, las procesa y devuelve una respuesta al cliente.

    Ejemplos:

    • Correo electrónico: el cliente (como Gmail u Outlook) se conecta al servidor de correo, el cual guarda y distribuye los mensajes. Cuando envías un correo, el cliente lo entrega al servidor, y cuando recibes, el cliente descarga el mensaje desde ese servidor.
    • Cuando escribes www.tienda.com en tu navegador, el cliente envía la petición al servidor, este busca los datos y responde con la página web que ves en pantalla.

  • Puertos de comunicación

    En esta comunicación entran en juego los puertos, que son canales virtuales para identificar diferentes servicios de red, gracias a estos puertos, el sistema sabe qué tipo de comunicación está manejando.

    • Puerto 80 (HTTP): tráfico web sin cifrado.
    • Puerto 443 (HTTPS): tráfico web seguro con certificado SSL/TLS.
    • Otros puertos comunes: 21 (FTP), 25 (SMTP), 3306 (MySQL).

  • Servidor Web

    • Un servidor web es el software (y hardware) encargado de recibir solicitudes del cliente y devolver los recursos necesarios: páginas HTML, imágenes, archivos CSS o respuestas JSON.

    Algunos de los más utilizados son:

    • Apache: Muy popular y flexible, soporta módulos para ampliar sus funcionalidades.
    • Nginx: Conocido por su alto rendimiento y capacidad para manejar muchas conexiones simultáneas.
    • Microsoft IIS: Integrado en sistemas Windows, ideal para entornos .NET.

    Sin un servidor web, los navegadores no podrían recibir la información que despliegan a los usuarios.

  • Funcionamiento de una Aplicación Web

    • El funcionamiento de una aplicación web se basa en la interacción constante entre el cliente (normalmente el navegador del usuario) y el servidor, a través de solicitudes y respuestas. El proceso puede describirse en pasos:

    1. El usuario accede a la aplicación desde su navegador o dispositivo móvil, ingresando una URL o haciendo clic en un enlace.
    2. El navegador envía una solicitud (request) al servidor web utilizando el protocolo HTTP o HTTPS.
    3. El servidor recibe la solicitud y la procesa. Si necesita información adicional, consulta a una base de datos (por ejemplo, la lista de productos, datos de usuarios o historial de compras).
    4. El servidor genera una respuesta (response), que puede ser una página HTML, un archivo JSON o cualquier recurso solicitado (imágenes, estilos, scripts).
    5. El navegador interpreta la respuesta y la presenta de manera visual al usuario.

    Ejemplo: Cuando entras a una tienda en línea y buscas un producto, el navegador envía tu búsqueda al servidor, este consulta la base de datos, obtiene los resultados y devuelve una página con los productos que coinciden con tu búsqueda.

  • Despliegue de Contenido Web
      • Desarrollo local: Creación y prueba en una computadora personal.
      Servidor de pruebas (staging): Un entorno para verificar que todo funcione bien.
      Producción: Publicación en un servidor web accesible al público.
      Dominio y DNS: Configuración de un nombre de dominio (ejemplo: www.miweb.com) que apunte al servidor.
      Mantenimiento: Actualizaciones, correcciones y mejoras periódica.

Conceptos clave de HTML5

HTML (HyperText Markup Language) es el lenguaje base que estructura el contenido de una página web. Con HTML5, se introdujeron etiquetas semánticas que mejoran la organización y accesibilidad del contenido.

Definiciones esenciales

  • Doctype: Es el preámbulo obligatorio que indica al navegador que debe seguir los estándares modernos y evitar el modo normal.<!DOCTYPE html>

  • Document Tree (DOM): Es la estructura jerárquica del documento HTML. Cada elemento es un nodo, y el DOM permite que JavaScript modifique contenido, estilo y estructura.

  • Etiquetas comunes de HTML:
    • Define un párrafo:<p>

    • Salto de línea:<br>

    • Línea horizontal:<hr>

    • Cita larga:<blockquote>

    • Cita corta:<q>

    • Abreviatura con tooltip:<abbr>

    • Título de obra:<cite>

    • Texto resaltado:<mark>

    • Texto menos importante:<small>

    • Énfasis fuerte:<strong>

    • Énfasis suave:<em>

    • Negrita sin semántica:<b>

    • Cursiva sin semántica:<i>

    • Subrayado:<u>

    • Texto tachado:<del>

    • Subíndice:<sub>

    • Superíndice:<sup>

    • Enlace con href:<a>

    • Imagen con src y alt:<img>

    • Página embebida:<iframe>

    • Lista con viñetas:<ul>

    • Lista numerada:<ol>

    • Lista de definición:<dl>

    • Contenedor genérico:<div>

  • Atributos: Modifican el comportamiento de los elementos.
    <img src="foto.jpg" alt="Descripción">

  • Comentarios: Ayudan a documentar el código sin mostrarse en la página.
    <!-- Esto es un comentario -->

  • Indentación: Organiza el código para que sea legible. No afecta la visualización, pero sí la claridad.

  • Sección <head>: Contiene metadatos, enlaces a estilos y scripts.
    <head>
      <title>Mi Blog</title>
      <meta charset="UTF-8">
    </head>

Organización semántica en HTML5

HTML5 promueve el uso de etiquetas que definen claramente el propósito de cada parte del contenido:

  1. Encabezado de página o sección:<header>

  2. Menú de navegación:<nav>

  3. Contenido principal:<main>

  4. Agrupación temática:<section>

  5. Contenido independiente:<article>

  6. Información complementaria:<aside>

  7. Pie de página:<footer>

Sobre CSS

Es un lenguaje que define el estilo de páginas web: colores, tamaños, posiciones y diseño.

Se puede aplicar en línea, en el head con <style> o en un archivo externo .css


Su propósito

CSS es el modelo de lenguaje caracteristico de html, diseñado principalmente para ser el proveedor del diseño del contenido de nuestro archivo html.
Su función es separar la estructura de la presentación, facilitando la creación de diseños web mas atractivos.


Sintaxis

elemento { propiedad: valor; }

Ejemplo: p { color: red; }


Indentación

Es un proceso simple que se sigue para la indentacion dentro de CSS, ya que todo se compone de el orden en el codigo.

Ejemplo CON Indentación:
p {
  color: red;
  margin: 20px;
  padding: 10px;
}


Comentarios

El metodo que se utiliza para dejar un comentario como programador en este lenguaje, es distinto a muchos otros. Pero funciona de la misma manera, no se muestra al usuario y solo lo ven las personas con acceso al codigo (o si eres curioso e inspeccionas la pagina con tu navegador)

/*COMENTARIO*/

Ejemplo de uso: /*ESTO ES UN COMENTARIO*/


Selectores principales

  • Tipo: h1, p, div
  • Clase: .caja
  • ID: #principal
  • Pseudoclases: :hover, :first-child

Modelo de caja

Cada elemento se compone de: contenido + padding + border + margin


Propiedades comunes

  • color, background, font-size, font-family
  • margin, padding, border, width, height
  • display: block, inline, flex, grid

Animaciones

  • transition: cambia suavemente un estilo
  • @keyframes: define animaciones personalizadas

Antecedentes de CSS3

CSS3 nació en 1999, introdujo nuevos módulos para la animación, las transformaciones y las transiciones, lo que permitió a los diseñadores web crear efectos más dinámicos y atractivos. Aunque antes de el existieron otras dos versiones, como CSS1 y CSS2.

CSS y sus versiones.

    CSS1
  1. Nace en 1996.
  2. Fue la primera versión de este lenguaje.
  3. Contaba con caracteristicas muy basicas para el diseño web.
    CSS2
  1. Nace en 1998.
  2. Amplió las caracteristicas editables permitidas. (Ej. tablas o formularios)
  3. Introdujo nuevos tipos de selectores, como selectores de atributo, pseudo-clases y pseudo-elementos.
    CSS3
  1. Nace en 1999.
  2. Añadió muchas nuevas maneras de editar el texto, cajas, contenido, sombras.
  3. Introdujo nuevos módulos para la animación, las transformaciones y las transiciones.

Hoja de estilo externa

Una hoja de estilo externa es un archivo aparte donde se programa todo el diseño de la web. La idea de separar el contenido HTML del estilo CSS nos ayuda a que nuestro código quede más limpio y fácil de entender.

    Ventajas
  1. El index principal queda más ordenado.
  2. Es más fácil actualizar, ya que solo cambias el CSS.
  3. Sigue sus propiar normas de indentacion.

Reglas en CSS

Una regla CSS está formada por un selector y un bloque de declaraciones. Cada declaración es una pareja propiedad: valor que define un estilo.

Ejemplo:

p { color: blue; font-size: 16px;}

Aquí el selector es p, y las declaraciones son color: blue; y font-size: 16px;.
